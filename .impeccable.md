# ShowDeck — Design Context

## Design Context

### Users
**Empresários de artistas, equipes financeiras e vendedores de shows.**
Usam o sistema no dia a dia para gerenciar agenda de apresentações, contratos, cachês e contratantes. O contexto de uso é profissional — geralmente desktop em escritório ou celular em trânsito. Precisam de informação densa mas legível, acesso rápido e zero fricção.

### Brand Personality
**Moderno · Direto · Confiável**

A interface deve transmitir profissionalismo sem ser fria. Pensa em uma ferramenta que um empresário musical de alto nível usaria sem vergonha em frente ao cliente. Eficiente como Linear, polido como Vercel, expressivo como Spotify.

### Aesthetic Direction
- **Referências**: Linear (densidade + produtividade), Vercel (contraste nítido + typography), Notion (content-first + whitespace), Spotify (dark mode excelente + hierarquia visual forte)
- **Tema**: Suporte completo a light e dark mode — o usuário escolhe. Dark mode deve ser de primeira classe, não um afterthought.
- **Cor primária**: Roxo `hsl(262 83% 58%)` — mantida, mas tokenizada via CSS variable para fácil personalização por organização.
- **Anti-referências**: Interfaces genéricas de ERP brasileiro, visual "planilha colorida", excesso de bordas/grades visíveis.

### Design Principles

1. **Densidade com respiro** — Informação densa como Linear, mas com espaçamento deliberado. Nunca sufocado, nunca vazio.

2. **Hierarquia antes de decoração** — Tipografia e peso fazem o trabalho. Cores e ícones reforçam, não substituem hierarquia textual.

3. **Dark mode de primeira classe** — Cada decisão de cor deve funcionar perfeitamente nos dois temas. Dark não é "inverter" — é redesenhar.

4. **Feedback imediato** — Toda ação tem resposta visual: hover states, loading states, toasts de confirmação. O usuário nunca fica sem saber o que aconteceu.

5. **Progressão natural** — O layout guia o olho: sidebar → header → conteúdo principal → ações. Sem elementos que competem por atenção.
